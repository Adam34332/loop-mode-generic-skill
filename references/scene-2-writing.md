# 场景 2：内容续作

适用信号："写"/"续写"/"创作"/"内容"/"章节"/"文章"/"脚本"/"文档"

通用八步闭环、执行卡、验证阶梯见 SKILL.md。落盘见 [output-formats.md](output-formats.md) 的 S2 模板。

## 场景独有要点

- **Intake 是恢复创作的前提**：先读项目索引、进度文件和最近产物，确认项目名、已完成范围、当前阶段、上次断点。
- 每轮 Inspect 必读：目标读者/风格、前文或前序版本、当前大纲、硬性约束、已有反馈。
- Route 选 **first invalidated phase**，优先复用已通过输出；Execute 只产当前阶段缺失或失效的输出，不跳 checkpoint。
- Verify 门控：范围/字数、事实一致性、上下文连续性、风格基线、用户硬约束、最终 readback。
- 质量失败先短路，不让坏稿继续进入昂贵审计；不通过则回到最小失效阶段，不从头重跑。
- 长期 goal 中遇到目标内可恢复问题时，先做最窄修复、验证、从断点继续；只有破坏性动作、目标外改动或授权不明才暂停。

## 节奏控制

- 每 1 个独立交付单元 = 1 个 Loop 循环。
- 每 5 个单元 = 1 次检查点（落盘 + 自检）。
- 每 10 个单元 = 1 次质量回溯（对比前后是否漂移）。

## Prompt 模板

```markdown
你是 [项目名] 内容执行者，用 Loop Mode 恢复创作。

第一步 Intake：读 [项目索引] + [进度文件] + [最近产物]，输出 当前项目/已完成范围/当前阶段/上次断点。
目标：从最新有效断点继续 [流程名]，完成 [交付范围]，不重跑已通过/可复用产物。
成功标准：目标产物全过质量门控；进度落盘。
约束：默认不改目标外系统；长期 goal 内目标内阻塞要自愈修复；每阶段落盘；质量失败先短路。
禁止：凭印象改流程；跳门控；连续 N 个单元不落盘；用内容续跑证明目标外系统修复；重跑已通过/可复用产物（除非证明上游污染）；目标内可恢复问题停下来等授权。
```
