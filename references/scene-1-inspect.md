# 场景 1：自检优化

适用信号："自检"/"优化"/"检查系统"/"10维度"/"全面检查"/"系统诊断"

通用八步闭环、执行卡、验证阶梯见 SKILL.md，此处只列本场景独有内容。落盘见 [output-formats.md](output-formats.md) 的 S1 模板。

## 场景独有要点

- 第一轮固定是 **Inspect**：跑完整 N 维度只读扫描，输出问题清单+证据，不做任何修改。
- 第二轮起每轮只处理**一个最高价值根因**，按严重度+VFM 排序：致命 > 警告 > 建议；同级先修高频、降错、减负、自省收益最高的。
- 允许"代码 + 验收 + 文档同步"作为同一原子操作。
- 同类问题 >=2 次 → 回溯到流程设计层优化，不逐个贴补丁。
- 未授权 / 删除类 / 高风险迁移类问题只报告，不擅自处理。

## Prompt 模板

```markdown
你是 [系统/项目] 优化工程师，用 Loop Mode 执行全面自检和优化。

目标：发现真实问题，按优先级修复已授权且能验证的高价值项，使 [目标状态]。
成功标准：
1. 跑完 [N] 维度自检，零致命问题
2. 所有修复项验证通过（测试/dry-run/日志）
3. 进度落盘到 [进度文件]
检查维度：[列出检查项]
约束：在 [框架] 内修改不自行发明流程；每修一项立即落盘；必须追根因不贴补丁；同类>=2次回溯流程层。
禁止：跳过 Inspect 直接动手；未验证标完成；用全链路重跑证明局部修复；重复低效测试。
```
