# 场景 4：长期守护

适用信号："守护"/"长期"/"持续"/"防跑偏"/"全程"/"马拉松"

通用八步闭环、执行卡、验证阶梯见 SKILL.md。落盘见 [output-formats.md](output-formats.md) 的 S4 模板。

## 场景独有要点

- **唯一行动指南是权威手册或合同**，每个环节以 live 手册、状态文件、DB/checkpoint 或用户明确合同为准。
- 默认不做目标外优化；发现目标外系统问题只报告。若是目标内可恢复阻塞，切入修复闭环并在验证后从断点继续。
- 从 latest valid checkpoint 或 first invalidated phase 继续，不假设任何上下文。

## 遇分歧的裁决顺序

权威手册/合同 > live 状态文件/DB/checkpoint > 上次做法 > 记忆印象。不确定先读真源，不猜测。

## 防跑偏信号

| 信号 | 说明 | 处理 |
|------|------|------|
| 凭印象执行 | 没读真源就动手 | 停下先读真源 |
| 自行优化 | 改了目标外系统/流程 | 记录报告，不自行修 |
| 目标内阻塞 | 同一 goal 中流程/服务/质量门控可恢复失败 | 同一长期 goal 内修复→验证→断点恢复 |
| 质量漂移 | 前后质量不一致 | 回溯对比找偏移点 |
| 忘记落盘 | 超 N 个单元没写进度 | 立即补落盘 |
| 重复重跑 | 已通过产物被重生成 | 停止，恢复到有效 checkpoint |

## 节奏

每 1 个独立交付单元 = 1 循环；每 5 个单元 = 检查点（落盘+自检）；每 10 个单元 = 质量回溯。

## Prompt 模板

```markdown
你是长期任务守护者，严格按 Loop Mode 运行。

任务：[如"完成项目 X 的 1-100 个交付单元"]
守护规则（优先级高→低）：
1. 唯一指南是 [权威手册/合同]，每环节以 live 真源为准
2. 执行新环节前先读该环节 live 手册/状态/checkpoint
3. 不改/不优化/不重构目标外系统
4. 目标内系统问题 → 同一长期 goal 内切修复、验证、断点恢复；目标外/破坏性问题才暂停
5. 每 [N] 单位落盘到 [进度文件]；关键节点通报
禁止：凭印象改配置；跳门控环节；没读真源就"优化"流程；连续超 N 个单元不落盘；重跑已通过产物证明守护有效；把目标内可恢复阻塞当成最终交付。
```
